Java desktop emulator

GameDuck

GameDuck is a desktop Game Boy and Game Boy Color emulation suite written in Java with a Swing frontend. It began as my final-year project at Lancaster University and has since grown into a feature-rich emulator focused on modern desktop UX, cheat support, portability, ROM library management, and visual customisation.

Build Breakdown

Why I built it

A dissertation project with room to grow

I built GameDuck as my final-year university project to explore low-level systems, emulator design, and desktop application architecture in a single ambitious codebase.

Stack

Java, Swing, Maven, and custom emulator logic

The project combines a Java emulation core with a Swing desktop frontend, file-backed helpers, library tooling, and a packaged runtime for straightforward Windows distribution.

Hard problems solved

Bridging emulator accuracy with real UX

The difficult part was not only emulating the hardware model, but also turning that into a usable product with save states, ROM management, patch handling, settings, and desktop-friendly workflows.

Lessons learned

Systems work needs product thinking too

GameDuck reinforced that technically interesting software still needs good tooling, clarity, and polish around it if you want other people to actually use it.

What's next

Compatibility and richer desktop features

The roadmap points toward broader hardware support, UI refinement, migration tooling, and more advanced emulator features such as link cable workflows and graphical enhancements.

What GameDuck Includes

ROM workflows

IPS patch loading without editing the source ROM

ROM hacks can be launched by layering an IPS patch over a base ROM at runtime, so the original file stays untouched and save data can remain separate.

Library

Managed game library with metadata and art

The app copies played ROMs into a local library, supports favourites, and caches box art, title screens, and screenshots from libretro sources.

Persistence

Save data, quick saves, and numbered state slots

Compatible cartridges can use battery-backed save data, while managed quick saves and save-state slots make it easier to suspend and resume play.

Customisation

Themes, palettes, controls, and shortcuts

Users can switch palettes, create custom colour presets, adjust themes, and configure both control mappings and application shortcuts.

Debugging

Boot ROM support and live serial output

If you have your own legal boot ROM dumps, GameDuck can use them and also exposes serial output plus on-disk debug logging for more technical workflows.

Project context

From dissertation project to desktop suite

GameDuck started life as my final-year project at Lancaster University, but the repository now goes well beyond a basic academic prototype. It combines the emulator core, the desktop UI, persistence helpers, metadata handling, save management, boot ROM tooling, and UI customisation in one codebase.

Desktop-first flow

The project is intentionally structured around an actual desktop user experience, with dedicated screens for browsing, configuration, and day-to-day play.

Portable release model

The packaged Windows build is designed to be easy to extract and run with no separate Java dependency for the end user.

Personal branding

The duck name and logo are a nod to Lancaster University and the campus ducks that inspired the project identity.

Codebase Structure

Backend / Emulation

The emulator core houses the CPU, instruction decoding, memory map, graphics pipeline, peripherals, timers, ROM loading, and mapper implementations.

Backend / Helpers

This layer handles save data, quick states, library persistence, metadata caching, artwork fetching, and the file-backed tools that sit around the core emulator.

Frontend

The Swing frontend owns the main window, display panel, menus, input routing, library browser, theme handling, and the overall desktop user flow.

Runtime data

The app manages local folders for the ROM library, cartridge saves, quickstates, cached artwork, and generated docs, while settings and boot ROM storage live in the user profile.

Roadmap Toward v1.0

Compatibility

Hardware coverage

Planned work includes Super Game Boy support and link cable emulation.

Usability

Cleaner updates and UI

The UI will see continuous refinement with a focus on clearer workflows, better feedback, and more intuitive controls, while the update process will be streamlined with a built-in updater and migration tools.

Advanced features

Link cable direction

Longer-term goals include two simultaneous emulations with link cable support, dual controller input, and consistent, tight, 60fps optimisation

Legal Notice

GameDuck does not ship with commercial ROMs or Nintendo boot ROM files.

Boot ROM support is available, but users need to supply their own legal dumps separately.

The project is a personal emulator suite and is not affiliated with Nintendo.
